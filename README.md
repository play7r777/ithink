# MacroForge

Кастомный макрос-автоматизатор. Идея — слить **NatroMacro** (пресет-движок
для Roblox-стиля автоматизации) и **TinyTask** (запись и проигрывание)
в один инструмент на **AutoHotkey v2**.

Пресет — это JSON: список шагов, настройки, hotkeys. Их можно записывать,
редактировать в GUI, открывать в любом текстовом редакторе и делиться ими
просто как файлами.

## Требования

- Windows 10/11
- [AutoHotkey **v2.0+**](https://www.autohotkey.com/) (v1 не поддерживается)

## Запуск

1. Установить AutoHotkey v2.
2. Двойной клик по `MacroForge.ahk`.
3. Слева выбрать готовый пресет (`Autoclicker`, `Roblox_WASD_Patrol`,
   `Conditional_ImageCheck`) или нажать **New**.
4. Нажать hotkey **Start** (по умолчанию `F1`) или кнопку **Start**.

Hotkey'и глобальные — работают, пока сфокусировано любое окно.
Дефолтный «паник»-hotkey — `Esc`: останавливает воспроизведение и
отпускает любые удержанные клавиши.

## Типы шагов

| Тип               | Назначение                                                          |
|-------------------|---------------------------------------------------------------------|
| `click`           | Клик мышью. Поля: `x`,`y`,`button`,`count` (все опциональны).       |
| `move`            | Перевести курсор в `x`,`y` со скоростью `speed`.                    |
| `drag`            | Зажать в `fromX,fromY`, тащить до `toX,toY`.                        |
| `scroll`          | Колесо вверх/вниз, `amount` тиков.                                  |
| `key`             | Тап клавиши (или удержание `duration` мс).                          |
| `keyDown`         | Нажать клавишу (без отпускания).                                    |
| `keyUp`           | Отпустить клавишу.                                                  |
| `send`            | Отправить текст/команды через `Send`/`SendText`/`SendInput`.        |
| `sleep`           | Пауза `ms` мс. Поле `jitter` (±%) — рандомизация для антибота.     |
| `loop`            | Повторить вложенные `steps` `count` раз (0 = бесконечно).           |
| `ifImage`         | ImageSearch по `image` в области; ветвление.                        |
| `ifPixel`         | Сравнение цвета пикселя; ветвление.                                 |
| `ifText`          | OCR-поиск текста на экране; ветвление. Поддерживает игры (Roblox).  |
| `ifWindow`        | Проверка: активно ли окно `title`; ветвление.                       |
| `ifVar`           | Условие по переменной; ветвление.                                   |
| `setVar`          | Установить переменную (`=`, `+=`, `-=`). Scope: local/global.       |
| `waitForImage`    | Ждать появления картинки до `timeout` мс (poll каждые `poll` мс).   |
| `waitForPixel`    | Ждать пикселя нужного цвета до `timeout` мс.                        |
| `loopWhileImage`  | Выполнять вложенные шаги, пока картинка видна.                      |
| `loopUntilPixel`  | Выполнять вложенные шаги, пока пиксель не станет нужного цвета.     |
| `loopWhileVar`    | Выполнять вложенные шаги, пока условие переменной истинно.          |
| `call`            | Вызвать другой пресет по имени.                                     |
| `webhook`         | Отправить POST-запрос с JSON-payload.                               |
| `label`           | No-op маркер (`name`), удобно в логе.                               |
| `log`             | Записать `message` в лог.                                           |
| `break`           | Выйти из текущего цикла.                                            |
| `focusWindow`     | Активировать окно по `title` (AHK WinTitle).                        |

### ifText — режимы захвата (`captureMode`)

Стандартный GDI-захват экрана **не видит DirectX-контент** (Roblox, игры). Используй поле **Capture Mode**:

| Режим     | Метод                              | Применение                                 |
|-----------|------------------------------------|--------------------------------------------|
| `window`  | PrintWindow + PW_RENDERFULLCONTENT | ✅ **По умолчанию.** Большинство игр (Roblox Windowed) |
| `dx`      | Windows.Graphics.Capture (DirectX) | ✅ Эксклюзивный Fullscreen, UWP-приложения |
| `desktop` | GDI BitBlt                        | ⚠️ Только рабочий стол и не-DX окна        |

**Дополнительные поля `ifText`:**
- **Window Title** — `ahk_exe RobloxPlayerBeta.exe` (пикер 🎯 Pick Window заполнит автоматически)
- **OCR Scale** — масштаб изображения перед OCR (1–3). Мелкий шрифт лучше читается при `scale: 2`
- **Region X1/Y1/X2/Y2** — ограничить область поиска на экране

**Пример для Roblox:**
```json
{
  "type": "ifText",
  "searchText": "Respawn",
  "captureMode": "window",
  "windowTitle": "ahk_exe RobloxPlayerBeta.exe",
  "ocrScale": 2
}
```

## Запись

`F4` (или кнопка **Record**) — старт/стоп записи. Программа слушает буквы
`a–z`, цифры `0–9`, типовые игровые клавиши (`Space`, стрелки,
`Shift/Ctrl/Alt`, `F1–F12`) и кнопки мыши. Позиция курсора сэмплируется
каждые 40 мс — прямые отрезки склеиваются в один `move`. После остановки
выскочит диалог «append / replace».

> AHK может биндить только ограниченный набор клавиш в pass-through режиме.
> Если нужна экзотика (мультимедиа, нестандартная раскладка) — добавь её
> в `lib/Recorder.ahk` → `_bindHotkeys()`.

## Hotkeys

Дефолт на пресет: `F1` Start, `F2` Stop, `F3` Pause/Resume, `Esc` Panic.
Переназначаются на вкладке **Hotkeys**, кнопка **Apply Hotkeys** применяет
без сохранения.

Синтаксис hotkey-строк AHK v2:

| Символ | Модификатор |
|--------|-------------|
| `^`    | Ctrl        |
| `!`    | Alt         |
| `+`    | Shift       |
| `#`    | Win         |

Примеры: `F1`, `^F1` (Ctrl+F1), `!s` (Alt+S), `+Esc` (Shift+Esc).

## Настройки

| Ключ              | Значение                                                       |
|-------------------|----------------------------------------------------------------|
| `speedMultiplier` | Делитель для каждого `sleep`. `2.0` — вдвое быстрее.           |
| `repeatCount`     | Сколько раз проигрывать весь список шагов. `0` = бесконечно.    |
| `startDelayMs`    | Задержка перед стартом (успеть alt-tab в нужное окно).         |
| `targetWindow`    | AHK WinTitle для `focusWindow` и опции `focusOnStart`.         |
| `focusOnStart`    | `1` активирует `targetWindow` перед воспроизведением.          |
| `stopOnError`     | `1` — остановиться на первом упавшем шаге. `0` — лог и дальше. |

## Структура проекта

```
MacroForge/
  MacroForge.ahk            <- запускать это
  lib/
    Constants.ahk           Cfg static class: типы шагов + пути
    JSON.ahk                парсер/сериализатор
    Logger.ahk              дневной файл + UI-sink
    PresetManager.ahk       load/save/normalize пресетов
    Player.ahk              исполнитель
    HotkeyManager.ahk       регистрация start/stop/pause/panic
    Recorder.ahk            запись ввода + синтез шагов
    StepEditor.ahk          модалки для шага и списка шагов
  presets/*.json            пресеты пользователя + примеры
  logs/YYYY-MM-DD.log       дневные логи
  images/*.png              картинки для шагов ifImage
```

## Советы по Roblox

- Дефолт `targetWindow` `ahk_exe RobloxPlayerBeta.exe` + `focusOnStart: 1`
  обычно достаточно.
- Если клавиша «залипла» — `Esc` (panic) принудительно всё отпустит.
- Не делись записями с личным текстом (чат и т.п.) — литералы попадают в JSON.

## Известные ограничения

- AHK v2 однопоточный. Пока макрос идёт, главное окно «висит», но hotkey'и
  всё равно отрабатывают.
- Запись курсора — 25 Hz. Очень резкие движения проиграются примерно.
- Ошибочные строки hotkey биндинга логируются и пропускаются, не падают.
- `ifImage` чувствителен к масштабу/DPI/теме. Обрезай тайтнее, повышай
  `variation`.
